提示词工程

Prompt Engineering
当前任务的指令 —— 定义目标、角色、格式、推理路径。
📥 输入内容 核心

SKILL “以资深数据科学家身份，用 Markdown 表格 对比三种聚类算法，并给出适用场景。”

⚡ 含 Skill（预设提示词包）
🎯 控制风格 · 结构 · 推理链
🧩

记忆工程

Memory Engineering
历史对话 / 偏好 —— 用户画像、长期偏好、过往事实。
📥 输入内容 持续累积

“用户 偏好 Python，上次项目使用 FastAPI，拒绝过 SQL 方案。延续该风格，引用之前 RAG 架构。”

🔁 跨会话 · 长期
📚 个性化 · 上下文连贯性
🔌

MCP 工程

Model Context Protocol
工具的描述与调用结果 —— 对接外部系统（搜索、数据库、API）。
📥 输入内容 工具增强

“调用 Google 搜索 MCP 获取最新 AI 新闻，工具返回：[摘要+链接]，基于此生成周报。”

🌐 对接外部系统
⚙️ 行动 · 实时数据 · 扩展能力
⚙️ 使用层 · 控制「模型怎么工作」
🔀

编排工程

Orchestration
流程设计 —— 多智能体拓扑、条件路由、状态管理、循环与终止。
⚙️ 控制方式 逻辑控制

“A生成方案 → B负责挑错 → C汇总；若A输出置信度 < 90%，触发联网搜索后重试。”

🧩 流程图 · 条件分支 · 迭代
📐 设计流程 · 状态流转 · 多智能体协作
🎛️

推理配置

Inference Config
计算参数 —— Temperature、Top-p、Frequency Penalty、Reasoning Effort。
⚙️ 控制方式 数值调优

“同一提示词‘写一首秋天的诗’，Temperature=0.2 每次雷同，Temperature=0.9 每次都不同。”

📊 随机性 · 长度 · 惩罚项
🧪 调参 · 控制确定性/创造性
🔄

模型路由

Model Routing
模型选择 —— 能力分级、成本/速度权衡、多模型并行、版本管理。
⚙️ 控制方式 选择控制

“复杂推理用 o1，日常对话用 GPT-4o-mini，创意写作用 Claude。同一问题同时发给两个模型，取优。”

🎯 选模型 · 并行 · 成本优化
⚖️ 谁来计算 · 质量/速度/成本权衡
💡 使用层面 · 完整动作：构造输入 + 控制系统
📥 输入层：提示词 · 记忆 · MCP ⚙️ 使用层：编排 · 推理配置 · 模型路由
⚡ 输入层决定「模型收到什么」 · 使用层决定「模型在什么流程、以什么参数、由谁来计算」
