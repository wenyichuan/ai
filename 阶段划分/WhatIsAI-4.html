输入层 + 流程层 + 计算层 + 资源层  —  哪些会消失，哪些是护城河
📥 输入层 · 构造「给模型什么」
输入层 ✍️

提示词工程

Prompt Engineering
当前任务的指令 — 目标、角色、格式、推理路径。
📥 输入 核心

SKILL “以资深数据科学家身份，用 Markdown 表格 对比算法。”

⏳ 1-2年消失 ❌ 非壁垒 模型理解力提升
💡 入门技能，不必深研
输入层 🧩

记忆工程

Memory Engineering
历史对话 / 偏好 — 用户画像、长期偏好、过往事实。
📥 输入 持续累积

“用户 偏好 Python，上次使用 FastAPI，拒绝过 SQL。”

⏳ 2-3年消失（长上下文） ❌ 非壁垒（个人级） 上下文窗口扩展
💡 个人级够用即可
输入层 🔌

MCP 工程

Model Context Protocol
工具描述与调用结果 — 对接外部系统（搜索、数据库、API）。
📥 输入 工具增强

“调用 Google 搜索 MCP，返回 [摘要+链接]，生成周报。”

🔒 模型无法内化外部系统 🏰 生态壁垒 工具生态网络效应
🔥 重点投入 · 连接世界
🔀 流程层 · 设计「模型在什么流程里工作」
流程层 📐

编排工程 Orchestration Engineering

多智能体拓扑 · 条件路由 · 状态管理 · 循环终止
业务流程是组织独有的 Know-How，模型学不会也拿不走。
⚙️ 控制方式 逻辑控制（流程图 / 代码）

“A生成方案 → B负责挑错 → C汇总；
若A输出置信度 < 90%，触发联网搜索后重试，最多3次。”

🔒 流程 = 管理经验 🏰 流程壁垒 越用越厚
🔥 重点投入 · 把业务逻辑固化为流程
🎛️ 计算层 · 控制「模型怎么计算」
计算层 🧪

推理配置 Inference Configuration

Temperature · Top-p · Frequency Penalty · Reasoning Effort
模型将内置“自适应Temperature”，根据任务类型自动调整。
⚙️ 控制方式 数值调优（滑条 / 参数）

“同一提示词‘写一首秋天的诗’，Temperature=0.2 每次雷同，Temperature=0.9 每次都不同。
o1 模型设置 Reasoning Effort=high 解决复杂数学。”

⏳ 1-2年消失 ❌ 非壁垒 自适应参数
💡 了解即可，未来自适应
🔄 资源层 · 选择「谁来计算」
资源层 ⚖️

模型路由 Model Routing

模型选择 · 多模型并行 · 成本/速度权衡 · 版本管理
模型越强，价格越分化。用最便宜的模型完成90%的任务，把昂贵的留给最关键的问题。
⚙️ 控制方式 选择控制（下拉 / 策略）

“复杂推理用 o1，日常对话用 GPT-4o-mini，创意写作用 Claude。
同一问题同时发给 GPT-4o + Claude，取最优答案。”

💰 成本工程 · 越强越省钱 🏰 成本壁垒 价格分化加剧
🔥 重点投入 · 降本增效的核心
💡 完整动作：构造输入 + 设计流程 + 调优计算 + 选择资源
