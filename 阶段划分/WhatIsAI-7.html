AI 使用的完整维度

输入 → 模型 → 输出  ·  还原为软件工程基本功
输入 → 模型 → 输出
任何 AI 应用逃不出这个公式 · 其余都是 辅助
📥 输入 · 拼字符串
✍️
提示词 = 字符串模板
Prompt · 当前指令
将用户意图、角色、格式约束拼成一段文本。
// 拼接字符串 prompt = f"你是一名{role}，用{format}格式回答：{user_input}"
✅ 必要 本质：字符串拼接
🧩
记忆 = 数据库查询 + 拼字符串
Memory · 历史偏好
从数据库/缓存中读出历史，拼进当前上下文。
// 查询 + 拼接 history = db.query("SELECT * FROM history WHERE user_id=?") prompt = f"{history}\n当前问题：{question}"
✅ 必要 本质：CRUD
🔌
MCP / 工具 = API 集成 + 拼字符串
Tool Calling · 外部系统
调用外部 API，把返回结果拼进上下文。
// 调用 API + 拼接 result = requests.get("https://api.weather.com") prompt = f"天气数据：{result}\n请据此生成周报"
✅ 必要 本质：API 集成
🧠 模型 · 调 API
🎛️
推理配置 = API 参数
Inference Params · Temperature / Top-p / Reasoning Effort
传参控制模型行为，就像调用任何 API 时传参数一样。
// 调用 API 时传参 response = openai.ChatCompletion.create( model="gpt-4o", messages=[{"role": "user", "content": prompt}], temperature=0.7, top_p=0.9 )
⚙️ API 参数 本质：传参
🔄 控制流 · 写 if-else / for / while
📐
编排 / 工作流 = 控制流
Orchestration · 多步 / 多智能体
用代码控制模型的调用顺序、条件和循环。
// 就是写代码 plan = generate_plan(prompt) # 调用模型 if confidence(plan) < 0.9: plan = refine(plan) # 再调一次 report = summarize(plan) # 再调一次
🏗️ 应用架构 本质：if-else / for / while
⚖️
模型路由 = 策略模式 / switch-case
Model Routing · 选哪个模型
根据任务复杂度/成本，决定调用哪个模型。
// 就是 switch-case if complexity == "high": model = "o1" elif complexity == "medium": model = "gpt-4o" else: model = "gpt-4o-mini" response = call(model, prompt)
🏗️ 应用架构 本质：策略模式

写控制流
if-else · for · while · 策略模式
