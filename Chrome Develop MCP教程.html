1 启动 Chrome 远程调试端口

首先需要以开启远程调试模式的方式启动 Google Chrome，以便外部工具（MCP Server）能够控制浏览器实例。

Windows 系统： 关闭所有 Chrome 窗口，然后按 Win + R 打开运行窗口，输入以下命令：

"C:\Program Files\Google\Chrome\Application\chrome.exe" --remote-debugging-port=9222 --user-data-dir="%TEMP%\chrome-debug-profile"

macOS / Linux： 终端执行（根据实际安装路径调整）

/Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome --remote-debugging-port=9222 --user-data-dir="$TMPDIR/chrome-debug-profile"
📸 [模拟截图] Chrome 启动后地址栏下方出现“DevTools 正在调试此浏览器”提示条 — 表示调试模式生效

✅ 验证调试端口是否就绪： 打开新终端执行以下命令，应返回浏览器版本信息 JSON。

curl http://localhost:9222/json/version
📸 [终端输出截图] 返回 {"Browser":"Chrome/xxx","webSocketDebuggerUrl":"..."} — 端口正常
💡 提示：若端口冲突可更换为 9223 等，后续 MCP 配置需同步修改 --browser-url 参数。
2 安装并配置 MCP 服务 (chrome-devtools-mcp)

MCP (Model Context Protocol) 服务作为桥梁，让 Claude 能够调用 Chrome DevTools 能力。使用 npx 快速启动。

2.1 创建 MCP 配置文件（Claude Code GUI 或 Claude Desktop 需要）
在用户目录下新建配置文件（若没有则创建）：

  • Windows: %APPDATA%\Claude\claude_desktop_config.json 或项目级 .claude/mcp.json
  • macOS: ~/Library/Application Support/Claude/claude_desktop_config.json

添加以下 MCP Server 定义：

{
  "mcpServers": {
    "chrome-devtools": {
      "command": "npx",
      "args": [
        "-y",
        "chrome-devtools-mcp@latest",
        "--browser-url=http://localhost:9222"
      ],
      "type": "stdio"
    }
  }
}

2.2 手动测试 MCP 服务可用性（确保 Node.js 已安装）

npx -y chrome-devtools-mcp@latest --browser-url=http://localhost:9222
📸 [命令行截图] 服务启动后显示 "MCP server running on stdio" 无报错
📌 注意：请保持 Chrome 远程调试窗口始终运行，MCP 需要附着到该浏览器实例上。
3 IDEA 集成 Claude Code GUI 插件

在 IntelliJ IDEA 中安装 Claude Code GUI 插件，获得 AI 对话式自动化测试能力。

  1. 打开 IDEA，进入 File → Settings → Plugins (macOS: IntelliJ IDEA → Preferences → Plugins)
  2. 搜索 "Claude Code GUI" 或 "Claude AI Assistant"，点击安装并重启 IDE。
  3. 配置 API 密钥：Settings → Tools → Claude Code，填入 API Key（可使用 DeepSeek API 或 Anthropic Key）。
  4. 若使用 DeepSeek，请修改 API Base URL 为 https://api.deepseek.com/v1，模型选择 deepseek-chat。
  5. 关联 MCP 服务器： 在 Claude Code 设置中找到 "MCP Servers" 配置项，指定上一步创建的 mcp.json 路径，点击刷新，看到 chrome-devtools 状态为 ● Connected 即成功。
📸 [IDEA截图] Claude Code 工具窗口显示 MCP 工具列表: navigate_page, click, fill, take_screenshot 等

现在您可以在 IDEA 右侧的 Claude 对话框中输入中文指令，例如：“使用 MCP 打开 https://example.com，截图并获取页面标题”，AI 将自动调用 Chrome DevTools 执行。

4 结合 DeepSeek 生成测试脚本与断言

利用 DeepSeek 大模型强大的代码生成能力，快速产出符合项目需求的自动化测试步骤（可直接用于 Claude Code 或 Playwright）。

典型工作流：

  • 在 Claude Code GUI 的对话框中切换至 DeepSeek 模型（或使用官方 API），输入上下文提示词。
  • 要求 DeepSeek 生成针对特定 Web 应用的测试计划，格式遵循 MCP 工具调用规范。
// DeepSeek 生成的测试动作序列示例
1. navigate_page("http://192.168.50.194:3103/#/login")
2. fill("input[name='username']", "admin")
3. fill("input[name='password']", "admin")
4. click("button[type='submit']")
5. wait_for("text=数据看板")
6. take_screenshot()
📸 [DeepSeek 网页对话截图] 提示词："生成数据中台登录模块的MCP自动化脚本" → 输出结构化动作

将生成的脚本直接粘贴到 Claude Code 对话框，即可执行并观察浏览器自动操作，极大提升测试开发效率。

⚠️ 重要注意事项（源自高级QA工程师经验）
  • 如果调用 MCP 访问网页失败（如超时、连接拒绝），请及时检查 Chrome 调试端口 9222 是否被占用/正确启动，并确保命令行参数无误。若持续失败，退出当前思考流程并反馈“MCP 连接异常，请检查浏览器远程调试状态”。
  • 部分 Web 应用的操作按钮可能是图标（无文字标签），交互时推荐使用 CSS 选择器或 aria-label 定位元素，可通过 MCP 的 get_page_structure 获取可访问性树辅助定位。
  • 建议在实际测试中开启控制台监听和网络请求捕获，便于发现隐藏 Bug。
