# 《认知画布》演讲脚本
## —— 面向 AI Agent 的数据基础设施：Connect → Understand → Operate

> **场景设定**：产品发布会 / 技术分享，时长约 8 分钟。
> **演示配合**：演讲者打开 `agent-data-infra.html`，屏幕初始是一个**完全空的环境**。
> **核心叙事线**：从"无数据"到"可投产的 Agent 数据基座"，跟着顶部步骤条一步步点出来。

---

## 第一幕：抛出问题（约 1 分钟）

【画面：空环境，顶部步骤条显示"0 · 空环境 · 尚未接入任何数据"】

各位好。先请大家看这块屏幕——现在是空的，没有数据、没有概念、没有接口。这正是今天绝大多数企业在引入 AI Agent 时的真实起点。

我们都听过这样的抱怨：
- "Agent 很智能，但它连我们数据在哪都不知道。"
- "好不容易接上数据，它又看不懂——把'覆盖率'算成了另一个口径。"
- "好不容易能查了，一放开权限，半夜把库跑爆了。"

问题不在模型，而在**数据基础设施**。今天我用 8 分钟，带大家从这个空环境出发，一步一步搭出一个让 AI Agent **找得到、看得懂、用得稳**的数据基座。

我们的方法论是三步：**Connect 连接、Understand 理解、Operate 操作**。本体论贯穿始终。

---

## 第二幕：Connect —— 先让 Agent "找得到"（约 1.5 分钟）

【点击"下一步" ×3，依次接入三个数据源】

**第一步，接入星链结构化遥测。**
【点击】看，左栏出现了第一个数据源：Iceberg 表，18 列、1240 万行，Schema 覆盖 96%。这是结构化数据——轨道根数、遥测时序。

**第二步，接入遥感影像——这是非结构化数据。**
【点击】注意这张卡片，它没有"行列"，是 86TB 的像素栅格。我们做了两件本体论上很关键的事：一是**资产目录化**，给每一景影像登记 footprint、分辨率、传感器波段；二是**AI 质检派生元数据**——云量、质量分，把"看不懂的像素"变成"可描述的资产"。

**第三步，接入地面站日志——半结构化。**
【点击】Kafka 上的 JSON 流，380 万条链路事件。

这一步我们用的工具是 **Apache Gravitino / Unity Catalog**。它的价值一句话：**统一技术元数据，消除数据孤岛**。现在三类截然不同的数据，进了同一个目录、同一套血缘。Agent 终于有了"地图"。

但请记住——**只有地图，Agent 还是看不懂地形。**

---

## 第三幕：Understand —— 让 Agent "看得懂"（约 2.5 分钟）

【点击"下一步"，切换到本体图谱视图】

**第四步，构建本体 TBox。**
看画布中央，AI 从三个数据源里抽取出了核心概念：**星链星座 → 轨道面 → 卫星**，**遥感影像**，**地面站**。绿色的边是已确认关系，置信度 0.95 以上。

这里要强调本体论的基本功：**TBox 定义概念、关系、约束；ABox 挂载实例**——1584 颗卫星、2.46 万景影像，都是 ABox 实例。概念和实例分离，语义层才能既稳定又鲜活。

【点击"下一步"】

**第五步，派生指标，并直面冲突。**
云量、信噪比、重访周期这些指标上线了。但注意画面下方这个红色节点——**"覆盖率"**。系统检测到它有**两种语义定义**：

- 定义 A，来自轨道仿真：**几何覆盖率**——卫星视场扫过就算，不管云量；
- 定义 B，来自影像质检：**有效观测覆盖率**——云量低于 20% 才算数。

【双击红色节点】弹出的裁决窗口，我建议选**"拆分为两个指标"**。这不是和稀泥——两个概念在业务上都真实存在，硬合并才是灾难。这就是本体论说的**概念消歧**：语义层是 Agent 与数据之间的"上下文合同"，歧义必须人工裁决，不能让 Agent 猜。

【点击"下一步"】

**第六步，匹配已验证查询。**
现在图谱上高亮的是"轨道面—地面站—信噪比"这条链路。右栏有一条**已验证查询**："轨道面 SNR 低谷与地面站天气关联"，置信度 92%。

什么是已验证查询？它是**经过沙箱多轮验证、被证明稳定正确的查询模板**。Agent 不需要每次从零写 SQL——高频意图，直接复用；低频新意图，才进入循环工程去验证。这是准确性的根本保障。

---

## 第四幕：Operate —— 让 Agent "用得稳"（约 2 分钟）

【点击"下一步"，切换到运营报表】

**第七步，执行一次真实的归因分析。**
这是刚才那次"P-07 轨道面 SNR 异常"任务的产出：**主因是批量姿态机动，次因是地面站雨衰**。注意顶部的"分析任务闭环"——提出意图 → 语义对齐 → 匹配查询 → 循环验证 → 产出结论 → 固化沉淀。六步，每一步都有依据，每一步都可回溯。

【点击"下一步"，回到图谱】

**第八步，护栏演练——这是我最想讲的部分。**
画面左上角红色节点是 **Agent 配额**。我把滑块推到 92%——

【现场拖动滑块，点"模拟触发"】

看，**API 网关限速**触发了，红色边在闪；配额到 95%，**任务调度器会暂停非关键任务**；如果 Agent 要访问敏感区影像，**OPA 策略引擎强制降采样**——精度降级，但绝不放行原图。而任何一次触发，都会**级联写入审计链**，通知合规审批人。

这就是为什么这套系统敢上生产：**45 天，零安全事故。** Agent 的能力越强，护栏就必须越硬。这不是限制创新，是让创新可持续。

【点击"下一步"】

**最后一步，固化沉淀。**
这次分析产生的新查询，已经被验证、被固化，回写进语义层，MCP 工具 +1。系统收敛度从 67% 提升到了 78%。

**这就是循环工程**：每一次 Agent 的使用，都让语义层更厚一点、更准一点。系统在**自我进化**。

---

## 第五幕：收束（约 1 分钟）

【画面：完成态，步骤条显示"完成 · 基座已可复用"】

我们回顾一下这 8 分钟走过的路：

| 阶段 | 动作 | 工具 | 结果 |
|---|---|---|---|
| **Connect** | 统一技术元数据 | Gravitino / Unity Catalog | 12 源、128 表，消除孤岛 |
| **Understand** | 构建业务语义层 | 本体 + 指标 + 已验证查询 | 42 概念、18 指标、23 查询 |
| **Operate** | AI 编排 + 循环工程 | Claude Code / Cursor + MCP + 护栏 | 8 工具、12.6k 调用/月、0 事故 |

三句话送给大家：

1. **没有 Connect，Agent 是盲人**——统一目录是它的眼睛；
2. **没有 Understand，Agent 是文盲**——语义层是它与数据之间的合同；
3. **没有 Operate，Agent 是脱缰的马**——编排让它跑起来，护栏让它不出事。

从空环境到可投产，我们点了不到十次。谢谢大家。
