AI 使用的三大维度

提示词 · 记忆 · MCP  —  所有输入的本质
✍️

提示词工程

Prompt Engineering
当前任务的指令 —— 用自然语言定义目标、角色、格式和推理路径。
📥 输入内容 核心

SKILL “以资深数据科学家的身份，用 Markdown 表格 对比三种聚类算法，并给出适用场景。”

⚡ 含 Skill（预设提示词包）
🎯 控制输出风格、结构、推理链 · Skill 是其封装形态
🧩

记忆工程

Memory Engineering
历史对话 / 偏好 —— 将用户画像、长期偏好、过往事实拼入上下文。
📥 输入内容 持续累积

“用户 偏好 Python，上次项目使用 FastAPI，拒绝过 SQL 方案。本次回答延续该风格，并引用之前讨论的 RAG 架构。”

🔁 跨会话 · 长期
📚 个性化 + 上下文连贯性
🔌

MCP 工程

Model Context Protocol
工具的描述与调用结果 —— 让模型使用外部系统（搜索、数据库、API 等）。
📥 输入内容 工具增强

“调用 Google 搜索 MCP 获取最新 AI 新闻，工具返回结果：[摘要+链接]，基于此生成周报。”

🌐 对接外部系统
⚙️ 行动 · 实时数据 · 扩展能力
💡 使用层面 · 唯一动作：构造输入
✍️ 提示词 = 当前指令 + Skill 🧠 记忆 = 历史偏好 🔌 MCP = 工具结果
⚡ 所有维度最终统一为「输入给模型的上下文」 — 无第四维
